9.1 Review: Iterators and Iterator Operations

iterator type is defined by each STL container class

iterator is assigned to a specific location in a container

     we can add an integer to vector and string iterators, but not to list iterators

contents of an entry referred to by an iterator are accessed using the dereference operator (*)
     *v_itr and *l_itr are l-values
     *s_itr is an r-value

stepping through a container is done using increment (++) and decrement operators (--)

we can change the container that a specific iterator is attached to as long as the types match
     as long as v and w are both vector<double>, code can be compiled

9.2 Additional Iterator Operation for Vector (& String) Iterators

initialization at a random spot in the vector
     v_itr = v.begin() + 1
     O(1) - constant time

these operations are not allowed for list operators
     requires a for loop to get to that point
     O(n) - linear time

9.3 The list Standard Library Container Class

both lists & vectors store sequential data that can shrink or grow

vectors are formed as contiguous array-like block of memory
lists are formed as a chain of separate memory blocks, connected by pointers
     not necessarily next to each other or in a particular order


lists have no subscripting operation
     only way to get to the middle of the list is to follow pointers one link at a time

STL lists have push_front and pop_front functions, in addition to push_back and pop_back
erase and insert are very efficient, independent of the size of the list

can't use the standard sort function
     must use the special sort member function defined by the STL list type
     STL list sort member function is just as efficient, O(n log n)

9.4 Erase & Iterators

STL lists and vectors have a special member function called erase
return value is a pointer pointing to the value after the value that has been erased
     the integer stored in the second entry of the list has been removed
     the size has shrunk by one
     iterator p doesn't refer to a valid entry
     iterator q refers to the item that was the third entry and is now the second

9.5 Insert

takes an iterator and a value and adds a link in the chain
     new value immediately before the item pointed to by the iterator
call returns an iterator that points to the newly added element

9.7 Working towards our own version of the STL list

underlying mechanism - linked lists
allows us to build custom classes that mimic the STL list class

9.8  Objects with Pointers, Linking Objects Together

two fundamental mechanisms of linked lists:
     creating objects with pointers as one of the member variables
     making these pointers point to other objects of the same type

9.9 Definition: A Linked List

definition is recursive
a linked list is either:
     empty
     contains a node storing a value and a pointer to a linked list

the first node is called the head node 
     the pointer to this node is called the head pointer
     pointer's value will be stored in a variable called head

9.10 Visualizing Linked Lists

head pointer variable is drawn with its own box
important to have a separate pointer to the first node
     first node may change

objects that have been dynamically allocated and stored are shown as boxes
     conceptual view only - these boxes could be allocated anywhere, not necessarily in order

last node must have NULL for its pointer value
